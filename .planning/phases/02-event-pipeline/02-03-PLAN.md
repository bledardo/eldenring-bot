<feature>
  <name>Express API Server for Watcher Events</name>
  <files>~/delu-bot/src/apiServer.js, ~/delu-bot/tests/apiServer.test.js</files>
  <behavior>
    POST /api/events with valid Bearer token and boss_encounter event → 200 {"ok": true}, fight stored
    POST /api/events with invalid token → 401 {"ok": false, "error": "Invalid API key"}
    POST /api/events with no Authorization header → 401 {"ok": false, "error": "Missing API key"}
    POST /api/events with tracking_elden_ring disabled → 503 {"ok": false, "error": "Tracking disabled"}
    POST /api/events with duplicate event_id → 200 {"ok": true, "duplicate": true} (idempotent, not stored twice)
    POST /api/events with boss_encounter → calls addEldenRingFight with correct args
    POST /api/events with player_death → calls addEldenRingFight with outcome "death"
    POST /api/events with boss_kill → calls addEldenRingFight with outcome "kill"
    POST /api/events with session_start → calls startEldenRingSession
    POST /api/events with session_end → calls endEldenRingSession
    POST /api/events with unknown type → 400 {"ok": false, "error": "Unknown event type"}
    GET /health → 200 {"ok": true} (no auth required)
  </behavior>
  <implementation>
Create ~/delu-bot/src/apiServer.js:

1. Create Express app (do NOT call app.listen in the module — export a startApiServer function that does).
2. Add express.json() middleware with 1MB limit.
3. Add auth middleware for /api/* routes:
   - Parse `Authorization: Bearer <key>` header
   - Look up Discord user ID via `storage.getEldenRingPlayerByApiKey(key)`
   - Attach `req.discordUserId` on success
   - Return 401 on failure
4. Add feature flag middleware for /api/* routes:
   - Check `features.isEnabled('tracking_elden_ring')`
   - Return 503 if disabled
5. Add `POST /api/events` route:
   - Extract `type`, `event_id`, `data`, `timestamp` from body
   - Dedup check: if `event_id` and `storage.isEldenRingEventSeen(event_id)` → return `{ok: true, duplicate: true}`
   - Mark event seen: `storage.markEldenRingEventSeen(event_id)`
   - Route by type to handler functions
   - Return `{ok: true}` on success
6. Add `GET /health` route (no auth).
7. Event handlers (separate functions):
   - `handleBossEncounter(discordUserId, data)` — addEldenRingFight with outcome from data or 'encounter'
   - `handlePlayerDeath(discordUserId, data)` — addEldenRingFight with outcome 'death'
   - `handleBossKill(discordUserId, data)` — addEldenRingFight with outcome 'kill'
   - `handleSessionStart(discordUserId, data)` — startEldenRingSession
   - `handleSessionEnd(discordUserId, data)` — endEldenRingSession
8. Export: `{ app, startApiServer, stopApiServer }`
   - `startApiServer()` — calls `app.listen(config.eldenRing.apiPort)`, returns server
   - `stopApiServer()` — calls `server.close()`
9. Schedule dedup cleanup every hour: `setInterval(() => storage.cleanEldenRingSeenEvents(), 60 * 60 * 1000)`

Use vitest for tests (delu-bot already has it in devDependencies). Tests should use supertest for HTTP assertions:
- Install: `cd ~/delu-bot && npm install -D supertest`
- Import app (not started server) for testing
- Mock storage and features modules
  </implementation>
