  <action>
Create `watcher/watcher.py` — the central detection loop that wires capture → detectors → FSM → events.

**Watcher** class:
- `__init__(self, config: Config, http_client: WatcherHttpClient, tray: TrayApp)`:
  - Initialize `ScreenCapture` with config.capture_fps
  - Initialize all detectors: `HealthBarDetector`, `YouDiedDetector`, `BossNameDetector`, `CoopDetector`
  - Initialize `BossFightFSM` with event callbacks wired to `self._on_encounter`, `self._on_death`, `self._on_kill`, `self._on_abandon`
  - Store http_client and tray references
  - `self._running = False`
  - `self._game_hwnd: int | None = None` — game window handle for focus detection

- `start(self, game_pid: int)`:
  - Find game window handle from PID using `win32gui.EnumWindows` (see research code example)
  - Set `self._running = True`
  - Send session_start event via http_client
  - Update tray to WATCHING
  - Enter `_detection_loop()`

- `stop(self)`:
  - Set `self._running = False`
  - Send session_end event via http_client
  - Cleanup capture resources
  - Update tray to NO_GAME

- `_detection_loop(self)`:
  Main loop running at target_fps. Each iteration:
  1. **Check game focus**: Use `win32gui.IsIconic(hwnd)` and `win32gui.GetForegroundWindow()` to check if game is minimized or not in foreground. If not focused → skip frame, sleep, continue. Log "Detection paused: game not in focus" once (not every frame).
  2. **Capture regions**: Capture BOSS_BAR_REGION. Also capture YOU_DIED_REGION and COOP_REGION as needed based on current FSM state (don't capture all regions in IDLE — save CPU).
  3. **Run detectors**:
     - Always check health_bar.detect(boss_bar_frame)
     - If FSM is in ACTIVE_FIGHT or FIGHT_RESOLVING: also run you_died.detect(you_died_frame)
     - If health bar just confirmed: run coop.detect(coop_frame) — only check co-op at encounter start
     - If encounter confirmed and no co-op: run boss_name.detect(boss_bar_frame) ONCE — cache result for duration of fight
  4. **Feed FSM**: Call `fsm.process_frame(boss_bar_detected=..., boss_name=..., death_detected=...)`
  5. **Debug screenshots**: If config.debug_screenshots, save the captured frame when any detection trigger fires (encounter, death, kill). Save to `{data_dir}/screenshots/{timestamp}_{event_type}.png`.
  6. **Frame pacing**: Calculate elapsed time, sleep remaining time to hit target FPS. Use `time.perf_counter()` for accurate timing.
  7. **Queue flush**: Periodically (every ~30s) try to flush event queue in case there are offline-queued events.

- **FSM event callbacks**:
  - `_on_encounter(self, boss_name: str)`: Log encounter. If co-op detected → log skip and return. Build event dict: `{"type": "boss_encounter", "boss_name": boss_name, "timestamp": iso8601}`. Send via http_client.
  - `_on_death(self, boss_name: str)`: Log death. Build event: `{"type": "player_death", "boss_name": boss_name, "timestamp": iso8601}`. Send.
  - `_on_kill(self, boss_name: str)`: Log kill. Build event: `{"type": "boss_kill", "boss_name": boss_name, "timestamp": iso8601}`. Send.
  - `_on_abandon(self, boss_name: str)`: Log abandon. Build event: `{"type": "fight_abandoned", "boss_name": boss_name, "timestamp": iso8601}`. Send.

**Boss kill detection logic (DETECT-08):**
Boss kill = health bar disappears after an active fight AND no death is detected within the phase_transition_window. This is already handled by the FSM from Plan 02 (FIGHT_RESOLVING → timeout without death = kill). The Watcher just needs to feed the FSM correctly each frame.

**Performance considerations:**
- Capture only needed regions per state (don't capture co-op region every frame)
- OCR runs ONCE per encounter, not every frame (expensive ~100-500ms)
- Frame pacing prevents CPU saturation
- Log frame processing time at TRACE level for performance debugging

**Alt-tab handling:**
- Use `win32gui` to detect if game is foreground and not minimized
- When game loses focus: pause detection loop (just sleep), update tray tooltip to "Paused — game not in focus"
- When game regains focus: resume detection, update tray back to WATCHING
- Avoid logging every paused frame — log state changes only

   - `WatcherHttpClient(config.api_url, config.api_key, queue)`
   - `TrayApp(on_quit=shutdown)` — same as before

3. Update `on_launch(pid)` callback:
   - Log "Elden Ring detected (PID: {pid})"
   - Start watcher in a background thread: `threading.Thread(target=watcher.start, args=(pid,), daemon=True).start()`
   - NOTE: Do NOT send session_start here — it is already sent inside `Watcher.start()` (Task 1). Sending it here would cause a double-send.
4. Update `on_close()` callback:
   - Log "Elden Ring closed"
   - Call `watcher.stop()`
   - Send session_end event
   - Update tray to NO_GAME
5. **Auto-update placeholder**: Leave a clearly marked insertion point (`# AUTO-UPDATE CHECK — wired by Plan 06`) before the process monitor loop. Plan 06 Task 3 will add the actual updater call here.
6. **Periodic queue flush**: Start a background thread that calls `http_client.flush_queue()` every 30 seconds. This handles events queued during network outages.
7. **Clean shutdown**: On quit (tray quit or KeyboardInterrupt):
   - Stop watcher
   - Flush remaining queue
   - Stop tray
   - Log total session stats (events sent, events queued)

**Threading model:**
- Main thread: pystray tray icon (via `run()` — blocking) or process monitor
- Thread 1: ProcessMonitor polling loop
- Thread 2: Watcher detection loop (created on game launch, joined on game close)
- Thread 3: Queue flush timer

Note: pystray on Windows requires `run_detached()` if NOT on main thread. Keep the process monitor on main thread and tray detached, OR put tray on main and monitor in thread. Choose whichever is more stable on Windows — pystray docs recommend main thread for `run()` but `run_detached()` works from any thread.

Recommended: Tray via `run_detached()` (background), process monitor on main thread (blocking), watcher in daemon thread per game session.
