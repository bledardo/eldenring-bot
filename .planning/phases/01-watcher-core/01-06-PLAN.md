**watcher/updater.py**: GitHub Releases auto-update checker.
- `GITHUB_REPO = "OWNER/elden-ring-watcher"` — placeholder, user will set.
- `CURRENT_VERSION` — read from `watcher.__version__`.
- `check_for_update(timeout: float = 5.0) -> dict | None`: Queries `https://api.github.com/repos/{GITHUB_REPO}/releases/latest`. Compares `tag_name` (stripped of "v" prefix) to CURRENT_VERSION using `packaging.version.Version` comparison. Returns `{"version": str, "download_url": str, "release_notes": str}` if update available, None otherwise. Catches all exceptions — must never crash the app.
- `download_and_replace(download_url: str) -> bool`: Downloads new .exe to a temp file alongside current exe. Creates a batch script (`updater.bat`) that:
  1. Waits 2 seconds for current process to exit
  2. Renames current exe to .old
  3. Renames new exe to current name
  4. Deletes .old
  5. Launches new exe
  6. Deletes itself
  Launches the bat script via `subprocess.Popen` and calls `sys.exit(0)`.
- `perform_update_if_available() -> bool`: Convenience function — check + download + replace. Returns True if update initiated (app will exit). Log all steps with loguru.

Create `.github/workflows/build.yml` — GitHub Actions workflow that builds the exe on every release tag.

**Workflow:**
```yaml
name: Build Watcher
on:
  push:
    tags: ['v*']  # Trigger on version tags

jobs:
  build:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python 3.11
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Install dependencies
        run: |
          pip install -r requirements.txt
          pip install pyinstaller

      - name: Download EasyOCR models
        run: |
          python -c "import easyocr; easyocr.Reader(['fr', 'en'], gpu=False)"

      - name: Build exe
        run: python build.py

      - name: Upload Release Asset
        uses: softprops/action-gh-release@v2
        with:
          files: dist/EldenWatcher.exe
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

**Key decisions:**
- Python 3.11 (not 3.12/3.13) — better PyInstaller compatibility with PyTorch
- Windows runner required — building Windows exe on Windows
- EasyOCR models downloaded before build — ensures they're bundled
- Release asset uploaded automatically on tag push
- Tag format: `v0.1.0`, `v0.2.0`, etc.
