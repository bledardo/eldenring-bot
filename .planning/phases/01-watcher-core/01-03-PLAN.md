  <name>Task 1: Build screen capture abstraction with resolution-adaptive regions</name>
  <files>watcher/capture.py</files>
  <action>
Create `watcher/capture.py` with a `ScreenCapture` class:

**CaptureRegion** dataclass: Defines a screen region as percentages (0.0-1.0) of screen dimensions. Fields: `left_pct`, `top_pct`, `right_pct`, `bottom_pct`. Method `to_pixels(screen_width, screen_height) -> tuple[int, int, int, int]` converts to pixel coordinates.

**Pre-defined regions** (module-level constants):
- `BOSS_BAR_REGION = CaptureRegion(0.20, 0.82, 0.80, 0.95)` — bottom center where boss health bar appears
- `YOU_DIED_REGION = CaptureRegion(0.30, 0.35, 0.70, 0.55)` — center screen where "YOU DIED" text appears
- `COOP_REGION = CaptureRegion(0.0, 0.0, 0.15, 0.30)` — top-left where co-op phantom icons appear

**ScreenCapture** class:
- `__init__(self, target_fps: int = 10)`: Tries to initialize BetterCam first. If BetterCam fails (import error, init error), falls back to mss. Logs which backend is active. Detects screen resolution.
- `self._screen_width: int`, `self._screen_height: int`: Detected from capture backend.
- `_init_bettercam(self) -> bool`: Creates `bettercam.create(output_idx=0, output_color="BGR")`. Returns True on success.
- `_init_mss(self) -> bool`: Creates `mss.mss()` context. Gets monitor info for resolution. Returns True on success.
- `capture_region(self, region: CaptureRegion) -> np.ndarray | None`: Captures a specific screen region. For BetterCam: use `camera.grab(region=pixels)`. For mss: use `sct.grab(monitor_dict)` and convert to numpy BGR array. Returns None if capture fails (minimized, etc.).
- `capture_full(self) -> np.ndarray | None`: Captures full screen. Used for debug screenshots.
- `get_resolution(self) -> tuple[int, int]`: Returns detected screen resolution.
- `cleanup(self)`: Releases capture resources.

**Important design decisions:**
- BetterCam is primary (handles exclusive fullscreen), mss is fallback (borderless windowed only)
- All regions are percentage-based — no hardcoded pixel positions
- Return None on capture failure (don't raise) — caller handles gracefully
- Log resolution at startup: "Screen capture initialized: {width}x{height} via {backend}"
- Do NOT start BetterCam in streaming mode (`.start()`) — use `.grab()` per-frame instead, to allow region changes between calls

  <action>
**watcher/event_queue.py**: Disk-persisted event queue for offline resilience.
- `EventQueue` class:
  - `__init__(self, queue_dir: Path)`: Sets queue directory (e.g., `~/.elden-watcher/queue/`). Creates dir if needed.
  - Events are stored as individual JSON files named `{timestamp}_{uuid}.json` in the queue directory. This avoids corruption from concurrent read/write to a single file.
  - `enqueue(self, event: dict) -> Path`: Writes event to a new JSON file. Returns the file path. Adds `queued_at` timestamp to event.
  - `peek(self, limit: int = 10) -> list[tuple[Path, dict]]`: Returns oldest N events (sorted by filename = chronological order) without removing them.
  - `dequeue(self, path: Path)`: Deletes the event file (marks as sent).
  - `count(self) -> int`: Returns number of queued events.
  - `is_empty(self) -> bool`: Returns True if no events queued.
  - All operations use file locks or atomic writes to prevent corruption.
  - Log queue size changes with loguru.

**watcher/http_client.py**: HTTP client integrated with event queue.
- `WatcherHttpClient` class:
  - `__init__(self, api_url: str, api_key: str, queue: EventQueue)`: Stores config, creates `requests.Session` with retry adapter (5 retries, 0.5s backoff factor, retry on 429/500/502/503/504).
  - `_make_session(self) -> requests.Session`: Creates session with `HTTPAdapter(max_retries=Retry(...))` mounted on https:// and http://.
  - `send_event(self, event: dict) -> bool`: Enqueues event to disk FIRST, then attempts HTTP POST to `{api_url}/events`. Headers: `X-API-Key: {api_key}`, `Content-Type: application/json`. On success (2xx): dequeue from disk. On failure: leave in queue, log warning, return False.
  - `flush_queue(self) -> int`: Attempts to send all queued events (oldest first). Returns count of successfully sent events. Stops on first failure (server may be down — don't burn retries on every queued event).
  - `is_connected(self) -> bool`: Returns True if last send was successful. Tracks `self._last_success: bool`.
  - POST body format: `{"type": event_type, "data": {...}, "timestamp": iso8601, "api_key": api_key}`
  - Set request timeout to 10 seconds.
  - Log all send attempts and failures with loguru.
