## Critical Integration Breaks

6 cross-phase integration breaks found by integration checker:

### BREAK 1: Boss Name Key Mismatch (CRITICAL — affects 18 requirements)

**Watcher** sends `boss_name` key. **API Server** reads `boss_canonical_name || boss`. Result: every boss resolves to `"Unknown Boss"`. All downstream features (embeds, stats, betting odds, leaderboard, boss artwork) are broken.

**Fix:** Change `watcher.py` to send `boss_canonical_name` instead of `boss_name`.

### BREAK 2: Event ID Never Generated (deduplication disabled)

**Watcher** never includes `event_id` in events. **HTTP client** falls back to empty string. **API server** dedup check is falsy — silently skipped. Network retries create duplicate fights, double bet resolutions, inflated stats.

**Fix:** Generate `uuid4()` for each event in `watcher.py`.

### BREAK 3: Session ID Never Generated (session tracking fragile)

**Watcher** never sends `session_id`. Both `session_start` and `session_end` store sessions with `id: undefined`. Single-session workaround via timestamp filtering, but multi-session scenarios will corrupt data.

**Fix:** Generate session UUID in `main.py on_launch()`, pass to all events.

### BREAK 4: Duration Seconds Never Sent (fight time always 0)

**Watcher** never includes `duration_seconds` in events. All fight records store `duration_seconds: 0`. Stats show 0 time, leaderboard time ranking is meaningless.

**Fix:** Track fight start time in FSM, include duration in death/kill events.

### BREAK 5: fight_abandoned Not Handled (400 error)

**Watcher** emits `fight_abandoned`. **API server** has no handler — returns 400. Event stays in disk queue permanently.

**Fix:** Add handler or remove the event emission.

### BREAK 6: durationSeconds Not Forwarded in emitEvent (kill embed shows 0s)

**API server** `handleBossKill` does not include `durationSeconds` in `emitEvent()` call. Even if BREAK 4 were fixed, kill embeds would still show "0s".

**Fix:** Add `durationSeconds` to the emitEvent payload.

## Root Cause Analysis

The 6 integration breaks share a common root cause: **the Watcher (Phase 1) and API Server (Phase 2) were developed with mismatched event payload contracts.** The Watcher sends `boss_name` while the API expects `boss_canonical_name`. Critical fields (`event_id`, `session_id`, `duration_seconds`) were defined in API server handlers but never populated by the Watcher. The `fight_abandoned` event was implemented in the Watcher but no corresponding handler was added server-side.

These are all fixable with targeted changes to `watcher/watcher.py`, `watcher/main.py`, and one line in `src/apiServer.js`.
