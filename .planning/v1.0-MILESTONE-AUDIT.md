## E2E Flow Verification

All 5 flows traced by integration checker — no breaks found.

| Flow | Path | Status |
|------|------|--------|
| Boss Detection → Discord | Watcher → HTTP POST → apiServer → storage + EventEmitter → notifier → embed with bet buttons | COMPLETE |
| Bet Placement | User clicks button → bot.js handler → storage.placeEldenRingBet → ephemeral reply | COMPLETE |
| Bet Resolution | Death/kill → notifier → closeEldenRingBet → wallet update → embed with results | COMPLETE |
| Session Summary | Game close → session_end → notifier → session fights query → summary embed | COMPLETE |
| Stats Commands | /er-stats, /er-bosses, /er-leaderboard → storage queries → Discord embeds | COMPLETE |

### Design Observations (Not Breaking)

1. **session_start emitted with no subscriber** — `apiServer.handleSessionStart()` emits to EventEmitter, but no notifier listener exists. Intentional: no Discord notification needed for session start. Storage writes correctly. (COMM-04 satisfied)

2. **fight_abandoned excluded from summary display** — Abandoned fights stored and counted as attempts, but excluded from kills/deaths totals in session summary. Design choice: abandons (left boss area) aren't deaths or kills.
