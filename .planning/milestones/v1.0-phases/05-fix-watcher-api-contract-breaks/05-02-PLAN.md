**BREAK 5 fix — Add handleFightAbandoned function:**

Add a new handler function near the other handler functions (before the EVENT_HANDLERS map):
```javascript
function handleFightAbandoned(discordUserId, data) {
  const bossName = data.boss_canonical_name || data.boss || 'Unknown Boss';
  storage.addEldenRingFight(discordUserId, bossName, {
    timestamp: data.timestamp || new Date().toISOString(),
    outcome: 'abandon',
    duration_seconds: data.duration_seconds || 0,
    boss_canonical_name: bossName,
    session_id: data.session_id || null,
  });
  // Intentionally no emitEvent — no Discord notification for abandoned fights
}
```

Add `fight_abandoned: handleFightAbandoned` to the EVENT_HANDLERS map.

**BREAK 6 fix — Add durationSeconds to handleBossKill emitEvent:**

In the `handleBossKill` function, find the `emitEvent('boss_kill', { ... })` call and add `durationSeconds: fight.duration_seconds` to the payload object. Use `fight.duration_seconds` (from the stored fight record), NOT `data.duration_seconds` directly — this ensures the embed value matches what was persisted.

**Important:** Do NOT add emitEvent to handleFightAbandoned. Fight abandoned has no Discord notification — the notifier has no handler for it, and the correct UX is silence when the player leaves a boss arena.
