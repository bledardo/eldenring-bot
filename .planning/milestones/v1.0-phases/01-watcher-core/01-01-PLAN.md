**watcher/process_monitor.py**: Game process lifecycle detection.
- `ProcessMonitor` class:
  - `__init__(self, game_process: str = "eldenring.exe", poll_interval: float = 2.0)`: Stores config, `self._pid: int | None = None`, `self._running = False`.
  - `find_game_pid(self) -> int | None`: Uses `psutil.process_iter(['name', 'pid'])` to find the game. Case-insensitive name match.
  - `start(self, on_launch: Callable[[int], None], on_close: Callable[[], None])`: Blocking loop that polls for game process. When game found and was not running → call `on_launch(pid)`. When game disappears and was running → call `on_close()`. Sleeps `poll_interval` between checks. Respects `self._running` flag.
  - `stop(self)`: Sets `self._running = False`.
  - Log all state transitions (game found, game lost) with loguru.

**watcher/tray.py**: System tray icon with color-coded status.
- `TrayStatus` enum: `WATCHING` (green), `NO_GAME` (yellow), `DISCONNECTED` (orange), `ERROR` (red).
- `TrayApp` class:
  - `__init__(self, on_quit: Callable)`: Creates pystray.Icon with name "EldenWatcher", tooltip "Elden Ring Watcher", and right-click menu with "Status: {status}" (disabled info item) and "Quit" item.
  - `set_status(self, status: TrayStatus)`: Updates icon color and tooltip. Creates a 64x64 PIL Image filled with the status color. Use a simple circle-on-transparent-background design so it looks like a status dot.
  - `run(self)`: Calls `self.icon.run()` — MUST be called from the main thread (pystray requirement on Windows).
  - `run_detached(self)`: Calls `self.icon.run_detached()` — runs tray in background thread so main thread is free for other work.
  - `stop(self)`: Calls `self.icon.stop()`.
  - Status colors: green=(0,200,0), yellow=(220,180,0), orange=(220,100,0), red=(220,0,0).

**watcher/main.py**: Entry point that wires everything together.
- `main()` function:
  1. Load config via `load_config()`
  2. Setup logging via `setup_logging(config)`
  3. Log startup banner with version
  4. Create `TrayApp` with quit callback that stops process monitor
  5. Create `ProcessMonitor` with config values
  6. Define `on_launch(pid)` callback: logs game detected, updates tray to WATCHING
  7. Define `on_close()` callback: logs game closed, updates tray to NO_GAME
  8. Start tray with `run_detached()` (background thread) — set initial status to NO_GAME
  9. Start process monitor loop (blocking on main thread) wrapped in try/except for clean shutdown
  10. On KeyboardInterrupt or quit: stop monitor, stop tray, log shutdown
- `if __name__ == "__main__": main()`

Important: The tray MUST run detached (background thread) because the process monitor loop needs to run on the main thread. pystray's `run_detached()` handles this on Windows.
